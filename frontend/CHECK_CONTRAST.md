# Проверка контрастности текста

## Быстрая проверка в браузере

### 1. Запустите dev сервер

```bash
cd frontend
npm run dev
```

Сервер запустится на `http://localhost:3000` (или другом порту, если 3000 занят)

### 2. Проверьте следующие страницы:

#### ✅ Страница продуктов (`/products`)
- **Заголовки таблицы**: "Код", "Название", "Тип", "Ед. изм.", "Действия" должны быть **темно-серыми и хорошо видны**
- **Данные в таблице**: текст должен быть **темным и читаемым**
- **Плейсхолдеры**: если есть поля ввода, плейсхолдеры должны быть **видимыми серыми**

#### ✅ Страница заказов (через Dashboard → Orders)
- **Заголовки колонок**: должны быть **темно-серыми** (`text-gray-800`)
- **Данные в строках**: должны быть **темными** (`text-gray-900`)

#### ✅ Страница задач (`/schedule` или через Dashboard)
- **Заголовки таблицы**: должны быть **хорошо видны**
- **Статусы задач**: должны быть **читаемыми**

#### ✅ Страница инвентаря (`/inventory`)
- **Заголовки**: "Продукт", "Локация", "Количество" и т.д. должны быть **темными**
- **Данные**: все числа и текст должны быть **хорошо видны**

#### ✅ Страница MRP (`/mrp`)
- **Заголовки таблицы**: должны быть **темно-серыми**
- **Данные консолидации**: должны быть **читаемыми**

#### ✅ DSIZ страницы
- `/dsiz/planning` - проверьте все формы и таблицы
- `/dsiz/shift-actualize` - проверьте поля ввода
- `/dsiz/master-data` - проверьте вкладки

### 3. Что проверить визуально:

1. **Заголовки таблиц**:
   - ✅ Должны быть **темно-серыми** (почти черными)
   - ✅ Должны быть **полужирными** (font-semibold)
   - ❌ НЕ должны быть светло-серыми или почти белыми

2. **Текст в ячейках таблиц**:
   - ✅ Должен быть **темным** (text-gray-900)
   - ✅ Хорошо читаться на белом/светло-сером фоне
   - ❌ НЕ должен быть светло-серым

3. **Плейсхолдеры в input полях**:
   - ✅ Должны быть **видимыми серыми** (text-gray-500)
   - ✅ Должны иметь достаточный контраст
   - ❌ НЕ должны быть почти белыми

4. **Текст в input полях** (когда пользователь вводит):
   - ✅ Должен быть **темным** (text-gray-900)
   - ✅ Хорошо виден при вводе

## Автоматическая проверка контрастности

### Вариант 1: Chrome DevTools (Lighthouse)

1. Откройте страницу в Chrome
2. Нажмите `F12` (открыть DevTools)
3. Перейдите на вкладку **Lighthouse**
4. Выберите **Accessibility**
5. Нажмите **Analyze page load**
6. Проверьте раздел **Contrast** - не должно быть ошибок

### Вариант 2: Расширение для браузера

**WAVE Web Accessibility Evaluation Tool**:
- Установите расширение для Chrome/Firefox
- Откройте страницу
- Нажмите на иконку WAVE
- Проверьте раздел **Contrast Errors**

**axe DevTools**:
- Установите расширение
- Откройте DevTools → вкладка **axe**
- Запустите сканирование
- Проверьте ошибки контрастности

### Вариант 3: Онлайн инструменты

1. **WebAIM Contrast Checker**: https://webaim.org/resources/contrastchecker/
   - Скопируйте цвет текста и фона
   - Проверьте контрастность

2. **Contrast Ratio Calculator**: https://contrast-ratio.com/
   - Введите цвета
   - Проверьте соответствие WCAG AA/AAA

## Проверка через код

### Проверить все файлы с таблицами:

```bash
# Проверить, что все заголовки таблиц имеют правильный цвет
cd frontend
grep -r "text-gray-500" src/components --include="*.tsx" | grep -i "th\|thead"
# Не должно быть результатов (или только в комментариях)

# Проверить, что заголовки используют text-gray-800
grep -r "text-gray-800" src/components --include="*.tsx" | grep -i "th\|thead"
# Должны быть результаты для всех таблиц
```

### Проверить стили плейсхолдеров:

```bash
# Проверить, что в index.css есть стили для плейсхолдеров
grep -A 5 "input::placeholder" src/index.css
# Должен быть виден стиль с color: rgb(107, 114, 128)
```

## Если что-то не видно:

1. **Очистите кэш браузера**: `Ctrl+Shift+R` (hard refresh)
2. **Проверьте консоль браузера**: `F12` → Console (не должно быть ошибок)
3. **Проверьте, что изменения сохранены**: перезапустите `npm run dev`

## Контакты для обратной связи

Если после проверки вы все еще видите проблемы с контрастностью:
- Укажите конкретную страницу
- Укажите конкретный элемент (заголовок таблицы, плейсхолдер и т.д.)
- Сделайте скриншот для наглядности
