## Вкладка "Стратегия"

**Статус:** В разработке  
**Дата начала:** 02.02.2026  
**Связь с бизнес-процессом:** Раздел 3 "Вкладка 'Стратегия'" в `BUSINESS_PROCESS_REFERENCE.md`

### Описание

Вкладка "Стратегия" предназначена для стратегического планирования производства на основе новых и изменённых заказов покупателей. Позволяет оценить потребность в ГП, отметить заказы к планированию и пересчитать план производства.

### Состав элементов

#### 1. Гант-диаграмма (основной элемент)

- **Компонент:** Используется тот же компонент `GanttChart`, что и на вкладке "Расписание" (`frontend/src/components/scheduling/GanttChart.tsx`).
- **Данные:** Отображаются **утверждённые** задания для РЦ из предыдущей сессии планирования.
- **Назначение:** Визуализация текущего утверждённого плана производства перед пересчётом.

**Статус:** ✅ Компонент существует, требуется интеграция для отображения утверждённых заданий.

---

#### 2. Кнопка "Обновить заказы покупателей"

- **Действие:** Загружает данные о заказах покупателей и выявляет новые и изменённые заказы.
- **Текущий этап (v1):** Загрузка данных из файла `.cursor/_olds/info/dataset_customer_orders.csv`.
- **Следующий этап (v2):** Загрузка данных из таблицы `manufacturing_orders` в базе данных (после реализации синхронизации с 1С).
- **Результат:** После нажатия выводятся два списка:
  - "Новые заказы"
  - "Изменённые заказы"

**Статус:** ⏳ Требуется реализация:
- Парсер CSV файла (v1) или API для получения заказов из БД (v2)
- Интеграция с `OrderComparisonService` для выявления изменений
- Обновление UI списков после загрузки

---

#### 3. Списки "Новые заказы" и "Изменённые заказы"

- **Формат:** На текущем этапе — обычные списки (таблицы).
- **Следующий этап:** Планируется реализация drag & drop для перемещения заказов.
- **Отображаемая информация для каждой ГП:**
  - Название ГП (продукт)
  - Количество
  - Дата выполнения (due_date)
  - Рассчитанный приоритет срочности (URGENT/HIGH/NORMAL/LOW)
- **Интерактивность:**
  - Рядом с каждой ГП есть кнопка **"Принять"**.
  - При нажатии на "Принять":
    - ГП выделяется визуально (например, фоном или галочкой)
    - Кнопка меняется на **"Отменить"**
  - При нажатии на "Отменить":
    - Выделение снимается
    - Кнопка меняется обратно на "Принять"

**Статус:** ⏳ Требуется реализация:
- Компонент списка с кнопками "Принять"/"Отменить"
- Состояние выбранных заказов (selected orders)
- Визуальное выделение принятых заказов

**Связь с существующим функционалом:**
- Использовать данные из `OrderChangesPage` (новые и изменённые заказы)
- Интегрировать с `OrderChangesList` или создать новый компонент для вкладки "Стратегия"

---

#### 4. Фильтры по приоритетам URGENT/HIGH/NORMAL/LOW

- **Тип:** Кнопки-фильтры с множественным выбором (можно выбрать несколько приоритетов одновременно).
- **Поведение:** При выборе одного или нескольких приоритетов в списках "Новые заказы" и "Изменённые заказы" остаются только ГП с выбранными приоритетами.
- **Визуализация:** Выбранные фильтры должны быть визуально выделены (например, активный стиль кнопки).

**Статус:** ⏳ Требуется реализация:
- Компонент фильтров с множественным выбором
- Логика фильтрации списков по приоритетам
- Визуальная индикация активных фильтров

**Примечание:** Приоритет срочности должен рассчитываться на основе данных заказа (due_date, количество и т.д.) или передаваться из backend при выявлении изменений.

---

#### 5. Кнопка "Пересчитать план"

- **Действие:** Запускает пересчёт плана производства для отмеченных (принятых) ГП из списков "Новые заказы" и "Изменённые заказы".
- **Алгоритм пересчёта:** Будет рассмотрен после одобрения прототипа вкладки (отложено на следующий этап).
- **Результат:**
  - В Гант-диаграмме появляются новые задания для РЦ для отмеченных ГП
  - Принятые позиции исчезают из списков "Новые заказы" и "Изменённые заказы"

**Статус:** ⏳ Требуется реализация:
- Интеграция с существующим API планирования (DSIZ Planning или аналог)
- Передача списка принятых заказов в API пересчёта
- Обновление Гант-диаграммы после пересчёта
- Удаление принятых заказов из списков

**Связь с существующим функционалом:**
- Возможно использование `DSIZMRPService` или создание нового сервиса стратегического планирования
- Интеграция с компонентом `DsizGanttChart` или `GanttChart`

### Технические детали

#### Используемые существующие компоненты

- `GanttChart` (`frontend/src/components/scheduling/GanttChart.tsx`) — для отображения утверждённого плана
- `OrderComparisonService` (`backend/core/services/order_comparison_service.py`) — для выявления новых/изменённых заказов
- API `GET /api/v1/manufacturing-orders/changes` — для получения списка изменений
- Возможно: `DsizGanttChart` для отображения результатов пересчёта

#### Новые компоненты для разработки

- `StrategicPlanningPage.tsx` — основная страница вкладки "Стратегия"
- `NewOrdersList.tsx` — список новых заказов с кнопками "Принять"/"Отменить"
- `ChangedOrdersList.tsx` — список изменённых заказов с кнопками "Принять"/"Отменить"
- `PriorityFilters.tsx` — компонент фильтров по приоритетам
- Backend API для пересчёта плана (если требуется новый endpoint)

#### Данные

- **Источник заказов (v1):** `.cursor/_olds/info/dataset_customer_orders.csv`
- **Источник заказов (v2):** Таблица `manufacturing_orders` в БД
- **Приоритеты:** URGENT, HIGH, NORMAL, LOW (enum или константы)
