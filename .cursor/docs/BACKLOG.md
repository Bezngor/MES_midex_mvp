## Стратегическое планирование (вкладка «Стратегия», таймлайн/Гант)

- **Зависимость длительности операции от объёма заказа:** Сейчас длительность операции на Ганте берётся только из маршрута (`route_operation.estimated_duration_minutes`), без учёта количества заказа (шт/кг). Добавить расчёт длительности в зависимости от объёма заказа (например, норма времени на единицу × количество), чтобы большие заказы получали пропорционально большую длительность на таймлайне. См. чек-лист `.cursor/docs/TIMELINE_DISTRIBUTION_CHECKLIST.md`, пункт 1.2.

## Расписание (Schedule)

- **Автообновление после выпуска/отмены выпуска:** после нажатия «Выпустить» или «Отменить выпуск» блок «Выпущенные заказы» и Гант должны обновляться автоматически, без принудительного обновления страницы (F5). Сейчас для отображения выпущенного заказа в списке выпущенных нужно обновить окно браузера вручную.
- **Фильтр в блоке «Выпуск заказов в производство»:** добавить фильтры по **Дате**, **Покупателю**, **Продукту**, чтобы пользователь мог сужать список заказов перед выпуском. (Требует поддержки полей «Покупатель» и «код заказа 1С» в модели заказа — см. TASK_CUSTOMER_DATA.md.)
