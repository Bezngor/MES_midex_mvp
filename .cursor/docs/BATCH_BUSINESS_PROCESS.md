# Цепочка бизнес-процесса: создание и запуск производственных партий (батчей)

## 1. Условия для создания партии

- **Продукт (product_id)** — обязателен. Указывается продукт типа BULK (масса: крем, паста и т.п.), для которого создаётся партия.
- **Количество (quantity_kg)** — обязательно, строго больше 0. Объём партии в кг.
- **Номер партии (batch_number)** — необязателен. Если не указан, генерируется автоматически в формате `BATCH-YYYYMMDD-HHMMSS-XXXXXXXX`.
- **Уникальность** — значение `batch_number` должно быть уникальным в системе.
- **Опционально**: рабочий центр, оператор, плановое время старта, локация хранения, родительский производственный заказ.

При создании партия получает статус **PLANNED** (запланирована).

---

## 2. Этапы и статусы партии

| Статус         | Описание |
|----------------|----------|
| **PLANNED**    | Партия создана, запланирована к производству. Ожидает запуска. |
| **IN_PROGRESS**| Партия запущена, производство идёт. Зафиксированы время старта и (при необходимости) рабочий центр/оператор. |
| **COMPLETED**  | Партия завершена. Зафиксированы время завершения и результат. |
| **FAILED**     | Партия завершена с ошибкой (брак, сбой и т.п.). |

---

## 3. Условия изменения статуса

### 3.1 Запуск партии (PLANNED → IN_PROGRESS)

- **Допустимый переход**: только из статуса **PLANNED**.
- **Действия**: установка статуса IN_PROGRESS, фиксация `started_at` (фактическое время старта).
- **Условие**: батч должен существовать и иметь статус PLANNED. Иначе возвращается ошибка (400 или 404).

### 3.2 Завершение партии (IN_PROGRESS → COMPLETED)

- **Допустимый переход**: только из статуса **IN_PROGRESS**.
- **Действия**: установка статуса COMPLETED, фиксация `completed_at` (фактическое время завершения).
- **Условие**: батч должен существовать и иметь статус IN_PROGRESS. Иначе возвращается ошибка.

### 3.3 Общий PATCH батча

- Через общий эндпоинт `PATCH /api/v1/batches/{batch_id}` можно обновить часть полей (статус, времена, оператор, рабочий центр и т.д.) без жёсткой проверки перехода. Использовать для корректировок и перевода в FAILED при необходимости.

### 3.4 Удаление партии

- Партию можно удалить по ID. Ограничения по статусу в текущей реализации не накладываются (удаление возможно из любого статуса).

---

## 4. Краткая цепочка (создание → запуск → завершение)

1. **Создание** — пользователь выбирает продукт (BULK), количество; при необходимости рабочий центр и др. Партия создаётся со статусом **PLANNED**.
2. **Запуск** — пользователь нажимает «Запустить» для партии в статусе PLANNED. Статус меняется на **IN_PROGRESS**, фиксируется `started_at`.
3. **Завершение** — пользователь нажимает «Завершить» для партии в статусе IN_PROGRESS. Статус меняется на **COMPLETED**, фиксируется `completed_at**.

После завершения партия может учитываться в остатках, genealogy и отчётах.
