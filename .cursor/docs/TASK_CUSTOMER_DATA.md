# Задача: данные по покупателю в заказах и в блоке выпуска

**Цель:** В блоке «Выпуск заказов в производство» (Расписание) показывать для каждой строки не только ГП, количество и срок, но и:
- **Наименование покупателя**
- **Код заказа покупателя из 1С** (если он отличается от текущего `order_number` — уточнить, один ли это код или нужны оба поля)

**Текущее состояние:**
- В модели `ManufacturingOrder` есть `order_number` (уникальный номер заказа), он отображается в списке выпуска.
- Поля «покупатель» / «код заказа 1С» в модели и в API заказов нет.

**Что сделать:**
1. **Backend:** Добавить в модель производственного заказа (или в связанную сущность) поля, например: `customer_name` (наименование покупателя), при необходимости `customer_order_code_1c` (код заказа в 1С), если он не совпадает с `order_number`. Обновить миграции, схемы, API (чтение/запись заказов).
2. **Загрузка из 1С / CSV:** При импорте заказов из 1С или из CSV заполнять эти поля (в dataset_customer_orders.csv есть колонка «Покупатель» — использовать для `customer_name`).
3. **Frontend:** В `ReleaseOrdersBlock` выводить наименование покупателя и код заказа 1С (если есть) рядом с номером заказа, чтобы пользователь мог однозначно выбирать, какой заказ выпускать.

**Приоритет:** по согласованию с заказчиком (улучшение UX при ручном выборе заказов к выпуску).
