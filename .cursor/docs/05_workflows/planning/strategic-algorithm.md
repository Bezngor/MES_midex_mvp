## Назначение документа

Данный документ фиксирует **алгоритм стратегического планирования производства** для вкладки "Стратегия" MES-приложения. Алгоритм реализует процесс пересчёта плана производства на основе принятых заказов покупателей.

**Принцип:** Алгоритм работает строго последовательно по приоритетам, обрабатывая сначала все заказы высшего приоритета, затем следующие по убыванию приоритета.

---

## Важное: последовательность обработки по приоритетам

Обработка строго последовательная по приоритетам:

1. **Сначала обрабатываются ВСЕ отмеченные ГП в статусе URGENT** (резервируются компоненты и время работы РЦ).
2. **Затем обрабатываются ГП в статусе HIGH** (резервируются из свободного остатка, оставшегося после резерва компонентов и зарезервированной работы РЦ для URGENT).
3. **Потом — NORMAL** (из остатков после URGENT и HIGH).
4. **И если что-то останется, то обрабатываем LOW** (из остатков после URGENT, HIGH и NORMAL).

**Важно:** Каждый следующий приоритет работает с остатками после предыдущих приоритетов. Эта последовательность может быть нарушена в некоторых случаях, но их будем разбирать на следующих итерациях. Пока работаем по этой простой и логически верной цепочке.

## Этап 1: Резервирование компонентов BOM

### Входные данные

- Список принятых заказов (отмеченные "Принять" из "Новые заказы" и "Изменённые заказы")
- Порядок обработки: **сначала все URGENT, затем все HIGH, затем все NORMAL, затем все LOW**

### Процесс (для каждого приоритета последовательно)

#### Шаг 1: Обработка всех заказов с приоритетом URGENT

Для каждого заказа с приоритетом URGENT (в порядке поступления):

**1. Декомпозиция по BOM:**
- Уровень ГП (готовый продукт)
- Уровень Массы (полуфабрикат, если требуется)

**2. Проверка и резервирование компонентов:**

**Уровень ГП:**
- Проверить доступность **всех** компонентов BOM ГП
- Если **всех хватает** → зарезервировать **все** компоненты ГП
- Если **не хватает Массы** → перейти к проверке Уровня Массы
- Если после проверки Массы **всё хватает** → зарезервировать **все** компоненты ГП (включая компоненты Массы)
- Если **чего-то не хватает** → **ничего не резервировать**

**Уровень Массы:**
- Проверить доступность **всех** компонентов BOM Массы
- Резервировать компоненты **только если всех хватает**
- Если хотя бы одного не хватает → **ничего не резервировать**

**Пример:**
- Есть короб и туба, но массы не хватает
- Проверяем BOM Массы: всех компонентов хватает
- Резервируем компоненты Массы
- Резервируем **все** компоненты Уровня ГП (короб, туба)

**3. Обработка ошибок резервирования:**
- Если компоненты ГП не могут быть зарезервированы:
  - Пометить ГП в списках (визуальная индикация)
  - Показать кнопку drill-down для просмотра недостающих компонентов (что и сколько не хватает)
  - Можно нажать "Отменить" для исключения из текущего планирования

#### Шаг 2: Обработка всех заказов с приоритетом HIGH

Повторить процесс из Шага 1 для всех заказов с приоритетом HIGH, используя **остатки компонентов после резервирования для URGENT**.

#### Шаг 3: Обработка всех заказов с приоритетом NORMAL

Повторить процесс из Шага 1 для всех заказов с приоритетом NORMAL, используя **остатки компонентов после резервирования для URGENT и HIGH**.

#### Шаг 4: Обработка всех заказов с приоритетом LOW

Повторить процесс из Шага 1 для всех заказов с приоритетом LOW, используя **остатки компонентов после резервирования для URGENT, HIGH и NORMAL**.

## Этап 2: Планирование операций производства

### Входные данные

- Список ГП, у которых все компоненты BOM зарезервированы (сгруппированные по приоритетам)
- Порядок обработки: **сначала все URGENT, затем все HIGH, затем все NORMAL, затем все LOW**

### Процесс (для каждого приоритета последовательно)

#### Шаг 1: Обработка всех ГП с приоритетом URGENT

Для каждого ГП с приоритетом URGENT (в порядке поступления):

**1. Проверка наличия Массы:**
- Проверить: Массы хватает на остатке?
- Если ДА → перейти к шагу 2
- Если НЕТ → перейти к шагу 3

**2. Если Массы достаточно:**
- Проверить наличие уже запланированного производства такого же ГП на дату/смену **не ранее той, на которую запланировано производство Массы**
- Если такое производство ГП уже существует → **объединить количество**
- Если не существует → зарезервировать ближайшую доступную дату/смену для РЦ

**3. Если Массы недостаточно:**
- Зарезервировать ближайшую доступную дату и смену для реактора (для производства Массы)
- После производства Массы → перейти к шагу 2

#### Шаг 2: Обработка всех ГП с приоритетом HIGH

Повторить процесс из Шага 1 для всех ГП с приоритетом HIGH, используя **свободное время РЦ и остатки компонентов после обработки URGENT**.

#### Шаг 3: Обработка всех ГП с приоритетом NORMAL

Повторить процесс из Шага 1 для всех ГП с приоритетом NORMAL, используя **свободное время РЦ и остатки компонентов после обработки URGENT и HIGH**.

#### Шаг 4: Обработка всех ГП с приоритетом LOW

Повторить процесс из Шага 1 для всех ГП с приоритетом LOW, используя **свободное время РЦ и остатки компонентов после обработки URGENT, HIGH и NORMAL**.

## Общие требования для Этапа 2

### 1. Расчёт времени изготовления

- **Обязательно** рассчитывать время изготовления на основании норм производства
- Использовать существующие нормы времени из справочников/конфигураций

### 2. Предотвращение пересечений по времени

- **Не допускать** пересечения операций на одном РЦ
- Следующая операция **не должна начинаться раньше**, чем завершается предыдущая
- Проверять занятость РЦ перед резервированием

### 3. Разделение операций между сменами

**Расписание смен:**
- **СМЕНА1:** с 8:00 до 20:00 текущего дня (12 часов)
- **СМЕНА2:** с 20:00 текущего дня до 8:00 следующего дня, но считается сменой текущего дня (12 часов)

**Правила разделения:**
- Если операция не может быть целиком исполнена в период одной смены → **разделить на две части**
- Первая часть: выполняется до конца текущей смены (до 20:00 для СМЕНЫ1)
- Вторая часть: выполняется в следующей смене с начала (с 20:00 для СМЕНЫ2)

**Пример правильного разделения:**
- СМЕНА1: Операция1 8 часов (с 8:00 до 16:00), Операция2.1 4 часа (с 16:00 до 20:00)
- СМЕНА2: Операция2.2 1 час (с 20:00 до 21:00) — продолжение Операции2

**Пример неправильного разделения:**
- СМЕНА1: Операция1 8 часов (с 8:00 до 16:00), Операция2 5 часов (с 16:00 до 21:00) — **неверно**, операция выходит за пределы смены (после 20:00)
- СМЕНА2: Операция2.2 1 час (с 8:00 до 9:00) — **неверно**, продолжение должно начинаться с 20:00, а не с 8:00

## Выходные данные

- **Зарезервированные компоненты BOM** (в inventory_balances или отдельной таблице резервирований) — с учётом последовательности по приоритетам
- **Запланированные операции производства** (ProductionTask или аналогичные записи):
  - Операции на реакторе (если нужна Масса)
  - Операции на РЦ (для производства ГП)
  - Объединённые операции (если ГП уже планировался)
  - Разделённые операции (если операция не помещается в одну смену)
- **Список ГП с ошибками резервирования** (для отображения в UI с drill-down)

## Связь с существующим функционалом

- **BOM Explode:** используется для декомпозиции заказов (уровень ГП и уровень Массы)
- **Inventory Balances:** проверка свободных остатков и резервирование (атомарное резервирование всех компонентов, с учётом последовательности по приоритетам)
- **MRP Service:** возможно использование логики расчёта потребности
- **DSIZ Planning:** возможно использование логики планирования операций по сменам и реакторам
- **Production Tasks:** создание задач для запланированных операций
- **Нормы времени:** использование для расчёта длительности операций
- **Расписание смен:** проверка доступности смен и разделение операций между сменами

---

## Реализация

Алгоритм реализуется в сервисе `StrategicPlanningService` (backend) и интегрируется с кнопкой "Пересчитать план" на странице `StrategicPlanningPage` (frontend).
